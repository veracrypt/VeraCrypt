文档 版本历史

发行说明

给使用VeraCrypt 1.17版本或更早版本创建卷的用户的注意事项：
为避免暴露您的卷是否包含隐藏卷，或者如果您在使用隐藏卷/操作系统时依赖似是而非的否认能力，那么您必须重新创建外部和隐藏卷，包括系统加密和隐藏操作系统，并丢弃VeraCrypt 1.18a版本之前创建的现有卷。

1.26.7 (2023年10月1日):

  • 所有操作系统:
    • 安全：创建卷时，确保XTS主密钥与辅助密钥不同
      • 由于随机数生成器的特性，此问题不太可能发生，但必须添加此检查以防止攻击
      • 参考：CCSS，美国国家安全局（NSA）第3页评论：https://csrc.nist.gov/csrc/media/Projects/crypto-publication-review-project/documents/initial-comments/sp800-38e-initial-public-comments-2021.pdf
    • 移除TrueCrypt模式支持。可以使用1.25.9版本挂载或转换TrueCrypt卷。
    • 完全移除RIPEMD160和GOST89算法。使用这些算法的旧版卷不再能被VeraCrypt挂载。
    • 为系统加密和标准卷添加对BLAKE2s作为新的伪随机函数（PRF）算法的支持。
    • 引入对EMV银行智能卡作为非系统卷密钥文件的支持。
      • 无需单独配置PKCS#11模块。
      • 不需要卡PIN码。
      • 从银行卡上的唯一编码数据生成安全的密钥文件内容。
      • 支持所有符合EMV标准的银行卡。
      • 可以在设置中启用（转到设置 -> 安全令牌）。
      • 由法国雷恩国立应用科学学院（Institut national des sciences appliquées de Rennes）的一组学生开发。
      • 有关团队和项目的更多详细信息，请访问 https://projets-info.insa-rennes.fr/projets/2022/VeraCrypt/index_en.html。
    • 在卷创建过程中覆盖现有文件容器时，将其当前大小添加到可用空间中
    • 增加科西嘉语支持。更新多个翻译。
    • 更新文档
  • Windows:
    • 正式地，最低支持版本现在为Windows 10。VeraCrypt可能仍可在Windows 7和Windows 8/8.1上运行，但不会在这些平台上进行主动测试。
    • EFI引导加载程序：
      • 修复了密码超时值处理中的一个漏洞，该漏洞导致其被限制为 255 秒。
      • 救援磁盘：如果原始 Windows 加载程序从磁盘中丢失，则使用其嵌入式备份来增强“启动原始 Windows 加载程序”功能。
      • 添加 Blake2s 算法，并移除 RIPEMD160 和 GOST89 算法。
    • 默认启用内存保护。在“性能/驱动程序配置”下添加选项，以便在需要时禁用它。
      • 内存保护会阻止非管理员进程读取 VeraCrypt 的内存。
      • 它可能会阻止屏幕阅读器（辅助功能支持）读取 VeraCrypt 的用户界面，在这种情况下可以禁用它。
      • 可以通过在“HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\veracrypt”下将注册表值“VeraCryptEnableMemoryProtection”设置为 0 来禁用它。
    • 添加进程缓解策略，以防止 VeraCrypt 被其他进程注入。
    • 对 RAM 加密实现进行了小幅度增强。
    • 修复了 Windows 11 22H2 下安全桌面的问题。
    • 实现对挂载部分加密的系统分区的支持。
    • 修复了在启用“清除加密密钥”选项（仅适用于系统加密情况）时，新设备插入的误报检测问题。

    • 在挂载按钮上添加下拉菜单，允许不使用缓存进行挂载。
    • 为大体积磁盘的原地加密过程中可能出现的对数级性能下降问题提供了可能的解决方案。
    • 让扩展器在进一步操作之前先检查文件是否存在。
    • 允许为生成的密钥文件选择大小单位（KB/MB/GB）。

    • 实现扩展器用户界面的翻译。
    • 用现代的 IFileDialog 接口替换旧的文件/目录选择 API，以提高与 Windows 11 的兼容性。

    • 移除对密钥文件扩展名的推荐，并更新文档以提及第三方文件扩展名的风险。

    • 高级用户：添加注册表设置以控制驱动程序内部加密队列，以便为SSD磁盘调整性能，并在高负载下获得更好的稳定性。
      • 在注册表项HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\veracrypt下：
        • VeraCryptEncryptionFragmentSize (REG_DWORD)：加密数据片段的大小，单位为KiB。默认值为256，最大值为2048。
        • VeraCryptEncryptionIoRequestCount (REG_DWORD)：并行I/O请求的最大数量。默认值为16，最大值为8192。
        • VeraCryptEncryptionItemCount (REG_DWORD)：并行处理的加密队列项的最大数量。默认值和最大值均为VeraCryptEncryptionIoRequestCount的一半。
      • 三元组（FragmentSize=512，IoRequestCount=128，ItemCount=64）是一组可提高某些SSD NVMe系统顺序读取速度的参数示例。
