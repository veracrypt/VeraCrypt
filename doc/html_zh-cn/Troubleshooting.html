故障排除

本节介绍了在使用VeraCrypt时可能遇到的常见问题的解决方案。
注意：如果您的问题未在此列出，可能会在以下部分列出：
  • 不兼容性
  • 已知问题与限制
  • 常见问题解答
确保您使用的是VeraCrypt的最新稳定版本。如果问题是由旧版本的VeraCrypt中的漏洞引起的，那么该问题可能已经得到修复。注意：选择 帮助 > 关于 以查看您使用的版本。

 

问题：
即使根据基准测试，我使用的密码算法速度高于硬盘速度，但对卷的读写操作仍然非常缓慢。
可能原因：
这可能是由干扰应用程序引起的。
可能的解决方案：
首先，确保您的VeraCrypt容器没有保留给可执行文件的文件扩展名（例如.exe、.sys或.dll）。如果有，Windows和防病毒软件可能会干扰容器并对卷的性能产生不利影响。
其次，禁用或卸载任何可能造成干扰的应用程序，通常是防病毒软件或自动磁盘碎片整理工具等。对于防病毒软件，通常在其偏好设置中关闭实时（访问时）扫描会有所帮助。如果不起作用，请尝试暂时禁用病毒防护软件。如果还是不行，请尝试完全卸载它，然后重新启动计算机。
问题：
VeraCrypt 卷无法挂载；VeraCrypt 报告 “密码错误或不是 VeraCrypt 卷”。
可能原因：
卷头可能已被第三方应用程序或故障硬件组件损坏。
可能的解决方案：
  • 您可以尝试通过以下步骤从嵌入在卷中的备份恢复卷头：
    1. 运行 VeraCrypt。
    2. 点击 选择设备 或 选择文件 以选择您的卷。
    3. 选择 工具 > 恢复卷头。
问题：
成功挂载卷后，Windows 报告 “此设备不包含有效的文件系统” 或类似错误。
可能原因：
VeraCrypt 卷上的文件系统可能已损坏（或者该卷未格式化）。
可能的解决方案：
您可以使用操作系统提供的文件系统修复工具尝试修复 VeraCrypt 卷上的文件系统。在 Windows 中，是 'chkdsk' 工具。VeraCrypt 提供了一种在 VeraCrypt 卷上使用此工具的简便方法：首先，制作 VeraCrypt 卷的备份副本（因为 'chkdsk' 工具可能会进一步损坏文件系统），然后挂载该卷。在 VeraCrypt 主窗口（在驱动器列表中）中右键单击已挂载的卷，然后从上下文菜单中选择 '修复文件系统'。
问题：
尝试创建隐藏卷时，其最大可能大小意外地小（外部卷上有比这多得多的可用空间）。
可能原因：
  1. 外部卷已格式化为 NTFS
  2. 碎片整理
  3. 簇大小太小 + 外部卷根目录中的文件/文件夹太多。
可能的解决方案：
与原因 1 相关的解决方案：
与 FAT 文件系统不同，NTFS 文件系统始终将内部数据精确地存储在卷的中间。因此，隐藏卷只能位于外部卷的后半部分。如果此限制不可接受，请执行以下操作之一：
  • 将外部卷重新格式化为 FAT，然后在其中创建隐藏卷。
  • 如果外部卷太大而无法格式化为 FAT，则将该卷拆分为几个 2 太字节的卷（如果设备使用 4 千字节扇区，则为 16 太字节的卷），并将每个卷格式化为 FAT。
与原因 2 相关的解决方案：
创建一个新的外部卷（碎片整理不是解决方案，因为它会对合理推诿性产生不利影响 – 请参阅 碎片整理 部分）。
与原因 3 相关的解决方案：
注意：以下解决方案仅适用于在 FAT 卷中创建的隐藏卷。
对外部卷进行碎片整理（挂载它，在 '计算机' 或 '我的电脑' 窗口中右键单击其驱动器号，点击 属性，选择 工具 选项卡，然后点击 '立即进行碎片整理'）。卷进行碎片整理后，退出 磁盘碎片整理程序，然后再次尝试创建隐藏卷。

如果这没有帮助，请通过按 Shift + Delete 删除外部卷上的 所有 文件和文件夹，而不是通过格式化（事先不要忘记为该驱动器禁用回收站和系统还原），然后尝试创建
在这个 完全为空的 外部卷上再次创建隐藏卷（仅用于测试目的）。如果即使现在隐藏卷的最大可能大小仍未改变，那么问题的原因很可能是扩展根目录。 如果您没有使用 '默认' 簇大小（向导的最后一步），请重新格式化外部卷，并在这次将簇大小保留为 '默认'。
如果这没有帮助，请再次重新格式化外部卷，并比上次少复制一些文件/文件夹到其根文件夹。如果还是没有帮助，请继续重新格式化并减少根文件夹中的文件/文件夹数量。如果这不可接受或者仍然没有帮助，请重新格式化外部卷并选择更大的簇大小。如果仍然没有帮助，请继续重新格式化并增大簇大小，直到问题解决。或者，尝试在 NTFS 卷中创建隐藏卷。
问题：
出现以下问题之一：
  • VeraCrypt 卷无法挂载。
  • 无法创建 NTFS VeraCrypt 卷。
此外，可能会报告以下错误：“该进程无法访问此文件，因为它正被另一个进程使用。”
可能原因：
这可能是由干扰应用程序引起的。请注意，这不是 VeraCrypt 的 bug。操作系统向 VeraCrypt 报告该设备已被某个应用程序独占锁定（因此 VeraCrypt 不被允许访问它）。
可能的解决方案：
通常，禁用或卸载干扰应用程序会有帮助，这些应用程序通常是防病毒实用程序、磁盘管理应用程序等。
问题：
在 VeraCrypt 引导加载程序屏幕中，我试图输入密码和/或按下其他键，但 VeraCrypt 引导加载程序没有响应。
可能原因：
您使用的是 USB 键盘（而非 PS/2 键盘），并且您的 BIOS 设置中禁用了 USB 键盘的预引导支持。
可能的解决方案：
您需要在 BIOS 设置中启用 USB 键盘的预引导支持。为此，请按照以下步骤操作：
重启您的计算机，在看到 BIOS 启动屏幕后立即按下 F2 或 Delete 键，然后等待 BIOS 配置屏幕出现。如果没有出现 BIOS 配置屏幕，请再次重启（重置）计算机，并在重启（重置）计算机后立即反复按下 F2 或 Delete 键。当 BIOS 配置屏幕出现时，启用 USB 键盘的预引导支持。通常可以通过选择以下选项来完成： 高级 > 'USB 配置' > '传统 USB 支持'（或 'USB 传统模式'） > 启用。（请注意，“传统”这个词实际上会产生误导，因为现代版本的 MS Windows 的预引导组件需要启用此选项才能允许用户进行交互/控制。）然后保存 BIOS 设置（通常通过按下 F10 键）并重启计算机。有关更多信息，请参考您的 BIOS/主板文档，或联系您计算机供应商的技术支持团队寻求帮助。
问题：
系统分区/驱动器加密后，计算机重启后无法启动（也无法进入 BIOS 配置屏幕）。
可能原因：
您计算机的 BIOS 存在 bug。
可能的解决方案：

请按照以下步骤操作：

  1. 断开加密驱动器的连接。
  2. 连接一个安装了操作系统的未加密驱动器（或者在该驱动器上安装操作系统）。
  3. 升级 BIOS。
  4. 如果还是没有帮助，请向计算机制造商或供应商报告此 bug。

或者

  • 如果 BIOS/主板/计算机制造商/供应商没有提供解决此问题的更新，并且您使用的是 Windows 7 或更高版本，并且驱动器上有一个额外的引导分区（其大小小于 1 GB），您可以尝试在不使用这个额外引导分区的情况下重新安装 Windows（以解决 BIOS 中的一个 bug）。
问题：
出现以下问题之一：
  • 在系统加密预测试期间输入预引导认证密码后，计算机死机（在显示消息 '正在启动...' 之后）。
  • 当系统分区/驱动器被部分或完全加密，并且自系统分区/驱动器加密过程开始后系统首次重启时，在您输入预引导认证密码后计算机死机（在显示消息 '正在启动...' 之后）。
  • 隐藏操作系统克隆完成并输入其密码后，计算机死机（显示“Booting...”消息后）。
可能原因：
计算机BIOS中的错误或Windows引导加载程序存在问题。
可能的解决方案：
  • 升级你的BIOS（有关如何操作的信息，请参考你的BIOS/主板文档，或联系计算机供应商的技术支持团队寻求帮助）。
  • 使用不同型号/品牌的主板。
  • 如果BIOS/主板/计算机制造商/供应商没有提供解决此问题的更新，并且你使用的是Windows 7或更高版本，并且驱动器上有一个额外的引导分区（其大小小于1GB），你可以尝试在不使用这个额外引导分区的情况下重新安装Windows（以绕过BIOS中的一个错误）。
  • 有另外两种已知的解决此问题的方法，需要有Windows安装盘：
    • 使用Windows安装盘启动计算机，选择修复计算机。选择“命令提示符”选项，打开后，输入以下命令，然后重启系统：
      • BootRec /fixmbr
      • BootRec /FixBoot
    • 删除位于驱动器开头的100MB系统保留分区，将其旁边的系统分区设置为活动分区（这两个操作都可以使用Windows安装盘修复选项中的diskpart实用工具完成）。之后，在Windows安装盘上重启后运行启动修复。以下链接包含详细说明： https://www.sevenforums.com/tutorials/71363-system-reserved-partition-delete.html
问题：
尝试加密系统分区/驱动器时，在预测试期间，VeraCrypt引导加载程序始终报告我输入的预启动身份验证密码不正确（即使我确定密码是正确的）。
可能原因：
  • “Num Lock”和/或“Caps Lock”键的状态不同
  • 数据损坏
可能的解决方案：
  1. 当你设置预启动身份验证密码时，记住“Num Lock”和“Caps Lock”键是开启还是关闭（根据制造商的不同，这些键可能有不同的标签，如Num LK）。注意：你可以在设置密码之前根据需要更改每个键的状态，但你需要记住这些状态。
  2. 当你在VeraCrypt引导加载程序屏幕中输入密码时，确保每个键的状态与你设置密码时的状态相同。
注意：有关此问题的其他可能解决方案，请参阅本章的其他部分。
问题：
当系统分区/驱动器被加密时，操作系统每隔5 - 60分钟会“冻结”约10 - 60秒（可能同时出现100%的CPU使用率）。
可能原因：
CPU和/或主板问题。
可能的解决方案：
  • 尝试在BIOS设置和Windows控制面板的“电源选项”中禁用所有与节能相关的功能（包括任何特殊的CPU增强暂停功能）。
  • 用不同的处理器（不同类型和/或品牌）替换当前的处理器。
  • 用不同的主板（不同类型和/或品牌）替换当前的主板。
问题：
挂载或卸载VeraCrypt卷时，系统崩溃（出现“蓝屏”错误屏幕或计算机突然重启）。
或
自从我安装了VeraCrypt，操作系统频繁崩溃。
可能的原因：
  • 第三方应用程序中的错误（例如，防病毒软件、系统“优化工具”、驱动程序等）
  • VeraCrypt中的错误
  • Windows中的错误或硬件组件故障
可能的解决方案：
  • 尝试禁用任何防病毒工具、系统“优化工具”以及任何其他类似的应用程序。如果问题仍然存在，尝试卸载它们并重启Windows。

    如果问题仍然存在，请在系统崩溃或重启后不久运行VeraCrypt并选择帮助 > '分析系统崩溃'。VeraCrypt将分析Windows在崩溃时自动创建的崩溃转储文件（如果有）。如果VeraCrypt确定第三方驱动程序中的错误可能导致了崩溃，它将显示驱动程序的名称和供应商（请注意，更新或卸载驱动程序可能会解决问题）。无论结果如何，你都可以选择向我们发送有关系统崩溃的重要信息，以帮助我们确定是否是VeraCrypt中的错误导致的。
问题：
在Windows 7/Vista（可能包括更高版本）上，Microsoft Windows备份工具无法用于将数据备份到非系统VeraCrypt卷。
原因：
Windows备份工具中的错误。
可能的解决方案：
  1. 挂载你要备份数据的VeraCrypt卷。
  2. 右键单击位于该卷上的文件夹（或在“计算机”列表中右键单击其驱动器号），并从“共享”子菜单中选择一个选项（在Windows Vista上，选择“共享”）。
  3. 按照说明将文件夹共享给你的用户账户。
  4. 在Windows备份工具中，选择共享文件夹（网络位置/路径）作为目标。
  5. 启动备份过程。
注意：上述解决方案不适用于Windows 7（可能包括更高版本）的入门版和家庭版。
问题：
在Windows Vista或更高版本的Windows的“计算机”窗口中，无法更改VeraCrypt卷中文件系统的标签。
原因：
Windows的一个问题导致标签仅被写入Windows注册表文件，而不是写入文件系统。
可能的解决方案：
  • 在“计算机”窗口中右键单击已挂载的卷，选择属性，并为该卷输入一个新的标签。
问题：
我无法加密分区/设备，因为VeraCrypt卷创建向导提示它正在使用中。
可能的解决方案：
关闭、禁用或卸载所有可能以任何方式使用该分区/设备的程序（例如，防病毒实用程序）。如果问题仍然存在，右键单击桌面上的“计算机”（或“我的电脑”）图标，选择管理 -> 存储 -> 磁盘管理。然后右键单击你要加密的分区，点击更改驱动器号和路径。然后点击删除和确定。重启操作系统。
问题：
创建隐藏卷时，向导报告无法锁定外部卷。
可能的原因：
外部卷包含一个或多个应用程序正在使用的文件。
可能的解决方案：
关闭所有正在使用外部卷上文件的应用程序。如果问题仍然存在，请尝试禁用或卸载你使用的任何防病毒工具，然后重新启动系统。
问题：
当访问通过网络共享的文件托管容器时，你会收到以下一条或两条错误消息：
“服务器没有足够的存储空间来处理此命令。”和/或
“没有足够的内存来完成事务。”
可能的原因：
Windows注册表中的IRPStackSize值可能设置得太小。
可能的解决方案：
在Windows注册表中找到IRPStackSize 键，并将其设置为更高的值。然后重新启动系统。如果该键在Windows注册表中不存在，请在HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters处创建该键，并将其值设置为16或更高。然后重新启动系统。有关更多信息，请参阅： https://support.microsoft.com/kb/285089/ 和 https://support.microsoft.com/kb/177078/
